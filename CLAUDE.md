## Project context

**10xPlantsInventory** ā€” an AI-vision plant cataloging app (greenfield, ~3-week MVP). Users photograph plants across multiple locations; an AI provider suggests species + care info; the app schedules watering/winterization reminders. See `context/foundation/prd.md` for the full spec.

Auth is **passwordless magic-link** (Supabase OTP ā€” there is no password sign-in), and the **core domain is already built**: locations, plants, the AI-vision suggestion endpoint, and Supabase Storage for photos (see `src/pages/api/` and `supabase/migrations/`). The **reminder/scheduling** features described in the PRD are **not yet built**.

## Architecture

Astro 6 SSR app with React 19 islands, Tailwind 4, Supabase auth, and shadcn/ui. Deployed to Cloudflare Workers.

### Rendering

`output: "server"` (astro.config.mjs) ā€” every route is server-rendered. `prerender` defaults to `false`; do not add `export const prerender = false` to API routes, it is redundant here.

### Auth flow

- `src/lib/supabase.ts` ā€” `createClient(headers, cookies)` builds a `@supabase/ssr` cookie-session client. **It returns `null` when `SUPABASE_URL`/`SUPABASE_KEY` are unset** ā€” every caller must null-check and degrade gracefully (the app is designed to run unconfigured).
- `src/middleware.ts` ā€” runs on every request, resolves the user, attaches it to `context.locals.user` (typed in `src/env.d.ts`). Redirects unauthenticated requests for any path in the `PROTECTED_ROUTES` array to `/auth/signin`. Add new protected paths there.
- API endpoints: `src/pages/api/auth/{signin,signout}.ts` ā€” `POST` handlers that read `request.formData()` and `context.redirect(...)` with errors passed back as `?error=` query params. `signin` sends a magic link via `signInWithOtp` (no password handler).
- Auth pages: `src/pages/auth/{signin,signup,check-email}.astro` (`signup` is a 308 redirect to `signin` ā€” magic-link only), plus `src/pages/auth/confirm.ts`, the GET handler that verifies the magic-link token (`verifyOtp`) and establishes the cookie session. `src/components/auth/*` holds the React form islands.

### Environment & secrets

`SUPABASE_URL` and `SUPABASE_KEY` are declared in the `astro.config.mjs` `env.schema` as `context: "server", access: "secret", optional: true`. Import them only via `astro:env/server`. Local config: `.env` for the Node toolchain, `.dev.vars` for the Cloudflare local runtime (both gitignored ā€” copy from `.env.example`). Local Supabase: `npx supabase start` (needs Docker).

## Conventions

- **Path alias**: `@/*` maps to `./src/*`.
- **Astro vs React**: Astro components for static content/layout; React components only where interactivity is needed. No Next.js directives (`"use client"` etc.).
- **Tailwind 4** is configured CSS-first via the `@tailwindcss/vite` plugin and `src/styles/global.css` ā€” there is no `tailwind.config` file.
- **Class merging**: use the `cn()` helper from `@/lib/utils` (clsx + tailwind-merge); never concatenate class strings by hand.
- **shadcn/ui**: "new-york" style, components in `src/components/ui/`. Add new ones with `npx shadcn@latest add [name]`. Icon library is `lucide-react`.
- **Lint is strict and type-aware** (`strictTypeChecked` + `stylisticTypeChecked`). `react-compiler/react-compiler` is set to `error` ā€” keep components compiler-safe (no mutation of props/state, follow the rules of hooks). Prefix intentionally-unused vars with `_`.
- **Placement**: services/helpers in `src/lib/`; shared types/DTOs in `src/types.ts`; React hooks in `src/components/hooks/`.
- **Supabase migrations**: `supabase/migrations/`, named `YYYYMMDDHHmmss_short_description.sql`. The core domain (`locations`, `plants`, `care_events`, and the `plant-photos` Storage bucket) is already migrated with RLS. Per the PRD's per-user isolation guardrail, enable RLS on every new table with granular per-operation, per-role policies.

## Tripwires

- The Cloudflare project name in `wrangler.jsonc` and the `name` in `package.json` are still `10x-astro-starter` (unchanged from the starter template).
- `context/archive/` is immutable ā€” never write there. `context/foundation/` holds the PRD, tech-stack, and shaping notes that drive this project.
- `CLAUDE.md.scaffold` is the original starter's rules file, kept for reference; the guidance in this section supersedes it.
- **AI-suggested fields are untrusted text.** `species`, `description`, `sunlight` (and any other free-text field returned by `/api/plants/suggest`) originate from the vision model and can carry attacker-controlled content via prompt injection in an uploaded photo. They are safe today only because React/Astro escape text by default ā€” never render them with `set:html` / `dangerouslySetInnerHTML`, and sanitize first if you ever must.

## 10xDevs AI Toolkit - Module 3, Lesson 4 (E2E Tests)

**For E2E tests, use the `/10x-e2e` skill.** It is the single source of truth
for the workflow ā€” risk ā†’ seed test + rules ā†’ generate ā†’ review against the five
anti-patterns ā†’ re-prompt ā†’ verify. The skill's `references/` carry the full
rules, anti-patterns, seed pattern, and prompt-template.

A few hard rules that hold even before you invoke the skill:

- **Locators:** `getByRole` / `getByLabel` / `getByText` first; `getByTestId`
  only when accessibility attributes are ambiguous. Never CSS selectors, XPath,
  or DOM structure.
- **Never `page.waitForTimeout()`.** Wait for state: `toBeVisible()`,
  `waitForURL()`, `waitForResponse()`.
- **Test independence + cleanup.** Each test runs standalone ā€” its own setup,
  action, assertion, and cleanup; unique ids (timestamp suffix) so parallel runs
  and re-runs don't collide.

Two boundaries to keep straight:

- **DOM (snapshot) is the default.** Vision (`--caps=vision`) is a supplement for
  visual-only risks (layout, z-index, animation); for pixel regression prefer
  deterministic tools (`toMatchSnapshot`, Argos, Lost Pixel). VLM model
  selection/cost is a debugging topic (Lesson 5), not testing.
- **Healer helps on selectors, harms on logic.** A changed selector ā†’ healer
  re-finds it (route through PR review). A changed business behavior ā†’ healer
  masks the bug; that failing-test-to-fix case is Lesson 5.
